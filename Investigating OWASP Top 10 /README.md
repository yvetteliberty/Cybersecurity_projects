# OWASP Top 10 (2021)
The OWASP Top 10 represents the most critical risks to web applications based on data from real-world applications.

1. A01: Broken Access Control
Improper enforcement of user permissions allowing unauthorized actions.

2. A02: Cryptographic Failures
Weak or improper use of encryption exposing sensitive data.

3. A03: Injection
Malicious input (e.g., SQL, OS, LDAP injection) executed by the application.

4. A04: Insecure Design
Flaws in application architecture and threat modeling.

5. A05: Security Misconfiguration
Default settings, unnecessary services, or exposed error messages.

6. A06: Vulnerable and Outdated Components
Using libraries or frameworks with known vulnerabilities.

7. A07: Identification and Authentication Failures
Weak authentication, session handling, or credential management.

8. A08: Software and Data Integrity Failures
Unsigned updates, insecure CI/CD pipelines, or untrusted sources.

9. A09: Security Logging and Monitoring Failures
Insufficient logging and alerting, delaying incident detection.

10. A10: Server-Side Request Forgery (SSRF)
Server fetches unintended resources due to manipulated requests.

- Proper authentication and authorization

- Secure configuration management

- Regular patching and updates

- Web Application Firewalls (WAF)

- Logging, monitoring, and alerting
